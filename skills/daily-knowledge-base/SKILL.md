# 每日经验知识库

与日报**分开**：日报给领导（精炼）；知识库给自己/团队长期查阅——**有理有据、数据支撑、逻辑脉络完整、可复现**。

## 核心原则（必须遵守）

1. **结论必有据**：每个判断标注证据类型——`[日志]` `[指标]` `[代码diff]` `[对话/操作记录]` `[用户口述]`。
2. **数据不编造**：材料中没有的数字写「待补充」，并列出需要补什么、从哪拿。
3. **逻辑链完整**：现象 → 假设 → 验证 → 证据 → 结论 → 行动，缺环要写明「未验证」。
4. **对比才有说服力**：改前/改后、轮次间、上午/下午，用表格或并列句呈现。
5. **失败同等重要**：失败轮次单独记录，写清与成功轮的**差异变量**。

## 收集材料（必须主动执行）

遵循 `remote-ssh-dev` → `remote-materials.md`：

1. **本机**：transcripts、terminals（含 ssh 输出）
2. **远端**（每台当日用过的 Host）：`git diff`、日志 tail、测试指标、`tmux` 历史、`ros2` 状态
3. 用户粘贴的远端日志、实机表
4. 若已有日报：`D:\Documents\工作汇报\日报\YYYYMMDD.md`

证据格式：`[远程] 主机=… | 路径=… | 时间=…`；指标表含**主机**列。
